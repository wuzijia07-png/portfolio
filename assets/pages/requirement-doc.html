1 需求背景

📖 业务场景

运营团队每周需上新50+ SKU到Amazon/Shopee等平台。目前Listing标题、五点描述、关键词、A+页面文案全靠人工撰写，单个Listing耗时约40分钟。随着SKU量增长，运营同学反馈工作量大、质量不稳定、多平台适配耗时严重。

😣 核心痛点

① 人工撰写效率低，新人上手慢；② 不同运营写的Listing质量参差不齐；③ 多平台（Amazon/Shopee/Lazada）格式要求不同，重复劳动；④ 缺乏历史数据参考，不知道什么样的Listing转化好

需求来源
运营部 · 张经理

2026年7月
影响范围
运营团队（约15人）

2周内原型验证

2 用户角色与使用场景

角色场景描述核心诉求
运营专员 每天需要为新品撰写Listing文案，在后台填写后发布到各平台 输入产品信息 → 一键生成多平台适配的Listing草稿 → 人工微调即可使用
运营主管 审核下属提交的Listing质量，抽查是否符合平台规范 查看生成记录、对比历史版本、追踪修改履历
产品经理 新品立项时提供产品参数和卖点，需要对接运营 上传产品规格表 → 自动生成Listing初稿 → 同步给运营优化

3 功能需求清单

序号功能模块功能描述优先级
F-01 AI Listing生成 输入产品名称、规格、卖点 → AI自动生成标题、五点描述、关键词、A+文案，支持Amazon/Shopee/Lazada多平台格式切换 P0 必须
F-02 产品信息导入 支持手动填写 / Excel批量导入 / ERP系统对接三种方式录入产品基础信息 P0 必须
F-03 文案在线编辑 生成结果可在页面内直接编辑修改，支持实时字符数统计和平台规范校验 P0 必须
F-04 历史记录 保存每次生成记录，支持查看、复制、对比历史版本 P1 重要
F-05 质量评分 AI对生成的Listing进行质量打分（完整性、关键词密度、吸引力等维度） P1 重要
F-06 数据看板 统计使用频次、生成数量、采纳率、各平台分布等运营数据 P2 锦上添花

4 验收标准

  • 输入产品名称+3个卖点后，30秒内生成完整Listing草稿（标题+五点描述+关键词）
  • 生成结果支持一键切换 Amazon / Shopee / Lazada 三种平台格式，各平台字符限制自动适配
  • 支持Excel批量导入，单次最多50个产品，批量生成时间不超过5分钟
  • 在线编辑器支持撤销/重做，关闭页面前自动草稿保存
  • 生成内容可复制、可导出为Excel，带时间戳和版本号
  • 原型需在2周内完成，可让运营同学实际操作并收集反馈

5 原型验证方案

使用 Claude Code + HTML/CSS/JS 搭建可交互原型，让运营同学实际操作验证需求理解是否准确。
